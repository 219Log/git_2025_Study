아파치 톰캣

  • 웹서버 정적데이터 처리에 적당하며 빠름
  • WAS동적인데이터를 처리, DB연결, 데이터 조작 처리에 활용

합쳐 부르는이유

  • 톰켓에서 편의를 위해 아파치기능(웹서비스 데몬, Httpd)를 포함하고 있음
  • 톰켓이 아파치 일부기능을 가져와서 제공해서 합쳐서부름

톰켓 파일구조

bin : 톰캣 실행에 필요한 실행,종료 스트립트파일 위치

conf : server.xml 및 서버 전체 설정과 관련한 톰캣 설정파일 위치

lib : 아파치와 같은 다른 웹서버와 톰켓을 연결해주는 jar

logs : 톰켓이 실행되는 동안 임시파일 위치

temp : 톰캣이 실행되는동안 임시파일 위치

webapps : 웹 어플리케이션 위치

work :jsp 파일을 서블릿형태로 변환한 java파일과 class 파일 위치

bin폴더

  • startup.bat : 톰캣을 실행(내부적으로 카타리나 뱃 실행)
  • shutdown.bat : 톰캣을 정지(내부적으로 카타리나 뱃을 실행)

conf폴더

  • server.xml : 서버 설정과 관련된내용
  • web.xml : 서버가 올라갈때 먼저 읽는 파일요한파일
  • context.xml : ???

webapps 폴더

  • 톰캣이 제공하는 웹애플리케이션 기본위치
  • 보통 이클립스통해서 생성하는데 폭더구조에 맞춰서 웹프로젝트 넣음

톰캣 사용방법

  1. zip 버전일때
    • 톰캣 경로로 들어간다음 cmd 실행
    • dir 입력후
    • startup.bat 실행
    • 실행하면
      • catalina_base
      • catalina_hame
      • jre_home
      • class pass

      경로가 나옴
      그리고 톰캣 실행창이 뜸

  1. 만약 창이 뜨자마자 꺼지거나 에러가나면 자바환경변수가 잘못된것
    • 잘못된경우
      • 환경변수 들어가서
        • JAVA_HOME
        • JRE_HOME 생성 및 수정

톰캣 포트 변경(server.xml)

  • 이클립스에서 설정한 포트와 충돌 나는것을 방지하기 위해 기본 톰캣 포트 변경할 수 있음
  • 기본적으로 port=”8005”인데 8105로 변경한다

웹홈페이지 경로변경(server.xml)

  • 웹 프로젝트를 톰캣의 webapps폴더가 아닌 다른경로로 변경하는 방법이다.
  • <host>태그 안에 빨간색으로 체크한 코드처럼 <context doc Base=””/>부분에 자신의 프로젝트가 있는경로를 추가하면된다.

톰캣 JDBC 연결 (context.xml)

  • JDBC를 이용해 자바와 오라클 데이터베이스를 연결해줘야 할 때 db정보를 입력해준다

  • 그리고 lib 폴더에 ojdbc 오라클 드라이버를 추가해주면 된다.
