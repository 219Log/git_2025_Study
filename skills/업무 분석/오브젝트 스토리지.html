오브젝트 스토리지

  • 오브젝트 스토리지는 파일 또는 블록 스토리지와 같은 기존 스토리지 시스 템과비교할때비교적새로운젶제품

  • 기존 파일스토리지의 확장성의 한계를 없앤 비정형데이터를 위한 스토리지

  • 스토리지란 컴퓨터에 대이터를 저장하는 저장소 역할을 수행하는 부품

  • 오즈젝트 스토리지는 데이터를 객체로 관리하는 기술
  • 모든 데이터는 파일이나폴더로 분할되지않고 여러 물리적 장치에 분산될수 있는 대형 저장소에 저장
  • 오브젝트 기반 스토리지를 보다 전통적인형태의 파일 및 블록스토리지와 비교하면 이해 가능

  • 파일 스토리지
    • 폴더에 데이터를 저장 계층적저장이라고 하는 방법
    • 종이문서가 저장되는 방식
    • 폴더구조로 경로를 사용하여 데이터를 찾음
    • 전송 프로토록 TCP/IP를 사용하고 장치는 리눅스에 NFS프로토콜 사용 윈도우는 SMB

  • 블록스토리지
    • 파일을 별도의 데이터를 블록으로 분할하고 이러한 블록을 별도의 데이터 단위로 저장
    • 각블록에는 주소가 있으므로 스토리지 시스템은 폴더 경로 없이도 데이터를 찾을 수있음
    • 더 작은 조각으로 분할 분산 방식으로 저장 가능
    • 파일 액세스할떄마다 스토리지 시스템 소프트웨어는 필요한 블록에서 파일을 조합
    • 블록프로토콜 FC 또는 iscsl을 사용하고 장치는 직접 연결 스토리지나 네트워크 작동

  • 오브젝트 스토리지
    • 파일 또는 오프젝트를 구성하는 데이터 블록과 해당 메타 데이터 모두 함께 보단됨
    • 각 오브젝트에 메타 데이터가 추가되므로 계층구조없이 데이터 엑세스 가능
    • 모든 오프젝트가 통합된 주소공간에 배치
    • 오브젝트를 찾기위해 사용자는 고유한 id를 제공함
    • 오브젝트 기반 스토리지는 tcp/ip를 사용 하며 장치는 http및 rext api를 사용하여 통신
    • 메타데이터는 오브젝트 스토리지의 기술의 중요한 부분.
    • 메타데이터는 사용자에의해 결정되며 기능과 특성에 따라 스토리지의 풀의 이미지를 유연하게 분석 검색 가능
    • 이점은 대형스토리 풀로 그룹화하고 이러ㅍ한 풀을 여러 위치에 분산 할 수 있음 무제한 확장과
      • 데이터의 복원력과 고가용성 향상

요약

  • 오브젝트 스토리지는 데이터 블록과 메타데이터를 보관함
  • 메타데이터로 계층구조없이 데이터 엑세스 가능
  • 오브젝트를 찾기위해 고유 id 제공
